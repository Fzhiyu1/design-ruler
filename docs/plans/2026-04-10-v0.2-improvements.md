## Task 2: measure selector 模糊匹配

**问题：** agent 执行 `design-ruler measure --selector ".global-header-lixiang"` 报错 `Element not found`。实际元素 class 名不完全匹配，agent 被迫 fallback 到 `webview_evaluate` 手写 JS 模糊查找。

**方案：** CLI 层支持 `~` 前缀语法表示模糊匹配。

```bash
# 精确匹配（默认）
design-ruler measure --selector ".dialog"

# 模糊匹配：class 包含 "global-header"
design-ruler measure --selector "~global-header"
```

**改动：**

Step 1: 新建 `src/engine/selector.ts`

```typescript
/**
 * Resolve selector: if starts with ~, convert to [class*="..."] fuzzy match.
 * Returns a standard CSS selector.
 */
export function resolveSelector(input: string): string {
  if (input.startsWith('~')) {
    const keyword = input.slice(1).trim()
    return `[class*="${keyword}"]`
  }
  return input
}
```

Step 2: 在 `measure.ts` 和 `screenshot.ts` 和 `overlay.ts` 的 buildXxxOptions 中调用 `resolveSelector`

Step 3: 测试
```typescript
describe('resolveSelector', () => {
  it('passes through normal selectors', () => {
    expect(resolveSelector('.dialog')).toBe('.dialog')
    expect(resolveSelector('#app')).toBe('#app')
    expect(resolveSelector('[data-id="x"]')).toBe('[data-id="x"]')
  })

  it('converts ~ prefix to class contains', () => {
    expect(resolveSelector('~global-header')).toBe('[class*="global-header"]')
    expect(resolveSelector('~card')).toBe('[class*="card"]')
  })
})
```

Step 4: CLI help 中说明 `~` 语法

## Task 3: measure 加 `--pick` 字段提取

**问题：** agent 频繁用 `measure | python3 -c "import json..."` 管道后处理 JSON，提取 bbox 或特定 CSS 属性。说明完整 JSON 输出对简单查询来说太冗长。

**方案：** `--pick` 参数，支持提取特定字段。

```bash
# 只输出 bbox
design-ruler measure --url "..." --selector ".box" --pick bbox
# → {"x":10,"y":20,"width":200,"height":100}

# 只输出特定 CSS 属性
design-ruler measure --url "..." --selector ".box" --pick padding,border-radius,gap
# → {"padding":"24px","border-radius":"12px","gap":"16px"}

# 完整输出（默认，不变）
design-ruler measure --url "..." --selector ".box"
```

**改动：**

Step 1: `src/commands/measure.ts` — `MeasureCommandOptions` 加 `pick?: string`

Step 2: `buildMeasureOptions` 解析 `raw.pick`

Step 3: 新增 `pickFields` 函数：
```typescript
function pickFields(result: MeasureResult, pick: string): string {
  if (pick === 'bbox') return JSON.stringify(result.bbox)
  if (pick === 'children') return JSON.stringify(result.children)
  // Otherwise treat as comma-separated CSS property names
  const props = pick.split(',').map(p => p.trim())
  const picked: Record<string, string> = {}
  for (const p of props) {
    if (result.computedStyle[p] != null) {
      picked[p] = result.computedStyle[p]
    }
  }
  return JSON.stringify(picked)
}
```

Step 4: `measure` 函数中，如果 `opts.pick` 存在则用 `pickFields` 输出

Step 5: CLI 加 `.option('--pick <fields>', 'Pick specific fields: bbox, children, or CSS property names (comma-separated)')`

Step 6: 测试 `pickFields` 的各种输入
